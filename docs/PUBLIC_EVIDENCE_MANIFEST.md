# Public Evidence Manifest

## Purpose

This manifest defines how public evidence is tracked for `FCCSecurity-Public`.

The repository is a public, defensive, local-first cybersecurity portfolio artifact. Evidence in this repository should support bounded claims about validation, remediation, residual risk, and public-release readiness. Evidence must not be treated as certification, external audit, or proof of absence of vulnerabilities.

## Evidence model

Preferred evidence types:

| Evidence type | Use |
|---|---|
| Commit SHA | Bind a change to immutable repository history |
| File path and line reference | Bind a claim to a specific artifact |
| GitHub Actions workflow run | Bind validation to CI execution |
| Generated report | Store scanner output in a reproducible format |
| SHA-256 hash | Verify integrity of generated evidence artifacts |
| Manual review note | Record human decision and residual risk |
| Issue or PR reference | Bind remediation and validation to workflow state |

## Regenerate evidence locally

Run from repository root:

```bash
python --version
python -m py_compile scripts/public_release_gate_check.py
python scripts/public_release_gate_check.py
```

Expected outputs:

```text
evidence/public-release-gate/public_release_gate_report.json
evidence/public-release-gate/public_release_gate_report.md
evidence/public-release-gate/SHA256SUMS
```

## Interpret SHA-256

SHA-256 hashes verify that a generated artifact has not changed since the hash was computed. They do not prove that the underlying review was complete, correct, or current.

Use the hash with:

```bash
sha256sum evidence/public-release-gate/public_release_gate_report.json
sha256sum evidence/public-release-gate/public_release_gate_report.md
```

On Windows PowerShell:

```powershell
Get-FileHash evidence/public-release-gate/public_release_gate_report.json -Algorithm SHA256
Get-FileHash evidence/public-release-gate/public_release_gate_report.md -Algorithm SHA256
```

## What this proves

This evidence can support that:

- a repository-local scanner exists;
- the scanner ran at a specific time;
- generated evidence files had specific hashes when recorded;
- selected public-release risk patterns were checked;
- no external targets were intentionally scanned by this workflow.

## What this does not prove

This evidence does not prove:

- absence of vulnerabilities;
- complete security;
- external audit completion;
- OpenAI endorsement, partnership, certification, or sponsorship;
- runtime security beyond the static scope reviewed;
- correctness of every public claim without human review.
